# design-sync notes — GINX Design System

Project: **GINX Design System** = `cfca63d7-2dfe-4b2a-a7c4-2ea4d7c98ce1`
Shape: `package`. Global: `window.GinxDS`. Entry: `./dist-designsystem/ginx-ds.es.js`.

## How this DS is built (important — it is NOT a published package)

The design system is a hand-authored folder inside the app repo (`src/design-system/`), not an npm
package. `npm run build:ds` orchestrates three steps and MUST be re-run before any converter build:

1. `vite build --config vite.ds.config.ts` → UMD/ES bundle (`GinxDS`) + scoped Tailwind CSS
   (`tailwind.ds.config.js`, content = `src/design-system/**`) into `dist-designsystem/`.
2. `tsc -p tsconfig.ds.json` → `.d.ts` tree into `dist-designsystem/types/` (drives component discovery).
3. `node scripts/ds-meta.mjs` → writes `dist-designsystem/package.json` (name `@ginx/design-system`,
   `types`, `module`) so the converter's entry walk-up resolves a named package + types root.

`vite build` uses `emptyOutDir: true`, so it wipes `dist-designsystem/` each run — the tsc + ds-meta
steps regenerate `types/` and `package.json` after it. Never point the converter at `dist-designsystem/`
without running the full `build:ds` first.

## Config gotchas

- `cssEntry` is resolved **relative to PKG_DIR** (= `dist-designsystem/`), so it is `./ginx-ds.css`,
  NOT `./dist-designsystem/ginx-ds.css`.
- `srcDir` is `../src/design-system` (PKG_DIR-relative) so src enrichment / JSDoc / grouping work.
- All 5 components land in group `general` (they sit directly in `src/design-system/`, no sub-dirs).

## Known render warns

- Card fired `[GRID_OVERFLOW]` because its preview frames are 340px wide → resolved with
  `cfg.overrides.Card: {"cardMode": "column"}`. Expected; not a new warn.

## Modal preview gotcha (fixed-overlay + transform ancestor)

The Modal is a `fixed inset-0` overlay. The preview mount has an ancestor with `transform: translateZ(0)`,
which makes `position: fixed` resolve against that ~80px containing block instead of the viewport → the
dialog lands at `top: -59px` and the title clips. Fix (in `previews/Modal.tsx`): wrap each story in a
sized `<div style={{ position:'relative', transform:'translateZ(0)', width:460, height:340, background:'#121212' }}>`
so the overlay centers inside a properly-sized containing block. Paired with
`cfg.overrides.Modal: {cardMode:"single", viewport:"460x340"}`. If you add more overlay/portal components,
they'll need the same wrapper.

## Re-sync risks (watch-list)

- The DS is generated from `src/design-system` source — always `npm run build:ds` before re-syncing;
  `dist-designsystem/` is gitignored and regenerated, so a stale one silently syncs old output.
- `dist-designsystem/package.json` is machine-generated by `scripts/ds-meta.mjs` — don't hand-edit it.
- Components assume a dark app background (`#121212`); previews inline that bg. If the palette changes,
  update `tailwind.ds.config.js`, `tokens.ts`, `ds.css`, AND `conventions.md` together.
- `conventions.md` names `gym-*` utility classes — only classes actually used by component source get
  compiled into the shipped CSS. `bg-gym-dark` is intentionally NOT claimed (no component uses it).
- Playwright/Chromium (playwright 1.61.1, chromium-1228) were installed in `.ds-sync/` for the render
  check; a fresh clone re-installs them.
